# 上位机程序 — 设计规格

## 1. 概述

有声书工具集的上位机 GUI 程序，用于调用 4 个 CLI 工具（scanner、organizer、transcriber、splitter），通过表格驱动的界面实现文件管理、批量处理和进度监控。

- **技术栈**: Tauri 2.0（Rust 后端 + 原生 Webview 前端）
- **前端**: 原生 HTML/CSS/JS（无框架，保持轻量）
- **通信**: 通过 `std::process::Command` 调用 CLI 子进程，解析 `--stream` JSON Lines 输出更新 UI

## 3. 功能规格

### 3.1 文件管理

- 支持拖拽文件/文件夹到窗口
- 支持点击"添加文件"和"添加文件夹"按钮
- 文件列表存储在 Rust 后端，前端通过 IPC 查询
- 每个文件/文件夹展开为单个文件行显示在表格中
- 支持视频文件拆分后生成子行

### 3.2 表格展示

| 列 | 说明 |
|----|------|
| 勾选 | 复选框，用于选择要操作的文件 |
| 文件名 | 文件名 + 图标（🎵音频/🎬视频），子行缩进显示时间段 |
| 大小 | 文件大小 |
| 元数据 | 标签形式显示：艺术家、专辑、时长等 |
| 拆分结果 | 视频文件：时间段标签（如 `00:00:00-00:05:00`）；音频文件：`─` |
| 识别结果 | Whisper 转写文字预览（截断显示）或状态 |
| 重命名预览 | 按模板渲染后的目标路径 |
| 进度 | 进度条 + 百分比 |
| 状态 | 圆点 + 文字：等待/进行中/完成/错误 |

### 3.3 操作方式

**右键菜单（单文件操作）：**
- 🔍 扫描元数据
- 📁 整理文件
- 🎤 语音转文字
- ✂️ 拆分音频（弹出时间段设置）
- ─────────
- 🗑 从列表移除

**底部按钮（批量操作）：**
- ▶ 执行选中：按流水线顺序执行所有操作
- ■ 停止：终止当前子进程
- 🔍 扫描：扫描选中文件的元数据
- 📁 整理：按模板重命名选中文件
- 🎤 转写：对选中文件进行语音识别
- ✂️ 拆分：对选中视频设置时间段并提取音频

### 3.4 流水线处理

选中文件后点击"执行选中"，按以下顺序处理：

```
video文件:
  [拆分] ──→ 为每个时间段生成子行 ──→ [转写] ──→ [扫描] ──→ [整理]

audio文件:
  直接 ──→ [转写] ──→ [扫描] ──→ [整理]
```

### 3.5 时间段设置

点击"拆分"时弹出对话框：
- 预设：按章节拆分 / 按固定时长拆分 / 自定义时间段
- 自定义：可添加多个 `开始-结束` 对
- 格式：`HH:MM:SS` 或秒数

### 3.6 进度监控

- CLI 子进程通过 `--stream` 参数输出 JSON Lines
- 后端逐行解析，通过 Tauri events 推送到前端
- 前端更新对应行的进度条和状态
- 日志区域实时追加日志行

## 4. Rust 后端 API

### Tauri Commands（前端调用后端）

| Command | 参数 | 返回 | 说明 |
|---------|------|------|------|
| `add_files` | `paths: Vec<String>` | `Vec<FileEntry>` | 添加文件/文件夹，展开为文件列表 |
| `remove_files` | `ids: Vec<u64>` | `()` | 从列表中移除文件 |
| `get_files` | — | `Vec<FileEntry>` | 获取当前文件列表 |
| `scan_metadata` | `ids: Vec<u64>` | `()` | 异步扫描元数据，通过 events 推送进度 |
| `transcribe` | `ids: Vec<u64>`, model, lang | `()` | 异步转写，通过 events 推送进度 |
| `split_video` | `id: u64`, segments | `()` | 异步拆分视频，通过 events 推送进度 |
| `organize` | `ids: Vec<u64>`, template, dest, dry_run | `()` | 异步整理，通过 events 推送进度 |
| `execute_pipeline` | `ids: Vec<u64>` | `()` | 按流水线顺序执行所有操作 |
| `cancel` | — | `()` | 终止当前子进程 |

### Tauri Events（后端→前端）

| Event | Payload | 说明 |
|-------|---------|------|
| `progress` | `{file_id, current, total, status}` | 文件进度更新 |
| `log` | `{level, message}` | 日志行 |
| `pipeline_done` | `{success, failed}` | 流水线完成 |
| `segment_added` | `{parent_id, segments}` | 视频拆分后新增子行 |

### FileEntry 数据结构

```rust
struct FileEntry {
    id: u64,
    path: PathBuf,
    kind: FileKind, // Audio | Video
    size: u64,
    metadata: Option<AudioMetadata>,
    segments: Vec<Segment>, // 拆分后的子项
    transcript: Option<Transcript>,
    rename_preview: Option<String>,
    progress: ProgressState,
    status: Status,
}

struct Segment {
    id: u64,
    label: String, // "00:00:00-00:05:00"
    start: f64,
    end: f64,
    temp_path: PathBuf,
    transcript: Option<Transcript>,
    progress: ProgressState,
    status: Status,
}
```

## 5. 前端组件

- `FileTable` — 主表格组件，渲染文件行和子行
- `DropZone` — 拖拽接收区（整窗口）
- `ContextMenu` — 右键菜单
- `BottomBar` — 底部操作按钮
- `SplitDialog` — 时间段设置弹窗
- `LogPanel` — 日志输出面板

## 6. 错误处理

- CLI 子进程异常退出 → 标记对应行为错误状态，日志记录 stderr
- 文件不存在/无法读取 → 添加时即报告错误
- 取消操作 → 发送进程终止信号，标记未完成任务为已取消
- 网络错误（模型下载）→ 日志提示重试

## 7. 数据持久化

- 无需持久化文件列表（会话级）
- 配置偏好（如模板、模型、语言等）可保存在 `app_data_dir()` 配置文件中
- 拆分后的临时音频文件存放在系统临时目录，退出时清理

## 8. 非功能需求

- 窗口大小：初始 1100×780，可调整
- 主题：深色主题
- 响应速度：UI 不阻塞，子进程异步执行
- 并发：同一时间只有一个流水线在执行
